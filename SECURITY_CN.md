# 安全策略

StructureClaw 是 AI 协同结构工程工作台。涉及本地执行、上传工程数据、凭据、运行工件的安全问题会被优先处理。

## 支持版本

安全修复面向当前活跃的 1.0 线。提交报告时，请注明测试使用的发布版本或 commit SHA。

## 报告漏洞

请不要用公开 issue 披露敏感安全问题。优先通过 GitHub Security Advisories 私下报告；如果不可用，再通过仓库所有者渠道联系维护者。

请包含：

- 受影响版本或 commit SHA
- 操作系统与运行模式（`npm`、源码目录或 Docker）
- 复现步骤
- 影响评估
- 已移除敏感信息的相关日志

## 密钥与本地数据

- 不要在报告附件中包含真实 API key、授权文件、数据库 dump 或私有模型数据。
- 请遮蔽 `LLM_API_KEY`、商业引擎授权信息和专有工程输入。
- 默认运行数据位于 `~/.structureclaw/`，除非通过 `SCLAW_DATA_DIR` 覆盖。

## 范围

范围内：

- 凭据泄漏
- 超出文档化 shell-tool 策略的任意命令执行
- 越过配置的 workspace/runtime 目录访问文件
- 上传文件处理问题
- 可能破坏用户数据或执行非预期代码的分析运行时行为

范围外：

- 不涉及安全边界的工程建议准确性问题
- 尚未支持某个商业引擎版本
- 需要 PKPM/YJK 厂商授权支持的本地授权失败
